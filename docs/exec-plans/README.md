# 実行プラン管理

## 概要

タスクの進捗・意思決定・学びを記録する場所。
エージェントが新しいコンテキストで作業を再開できるよう、常に最新の状態を保つ。

---

## ディレクトリ構成

```
exec-plans/
├── README.md           ← このファイル
├── active/             ← 進行中のプラン
├── completed/          ← 完了したプラン
└── tech-debt-tracker.md ← 技術的負債リスト
```

---

## プランのライフサイクル

```
作成（active/）→ 進行中（active/）→ 完了（completed/）
```

---

## プランのフォーマット

```markdown
# [タスク名]

## 目的
何を達成するか

## 完了条件
- [ ] 条件1
- [ ] 条件2

## 実装方針
どうやるか

## 進捗ログ
- 2026-03-22: 着手
- 2026-03-23: XX完了、YYで詰まった

## 学び・メモ
気づいたこと、次回への注意点
```

---

## ルール

- タスク開始時に必ず `active/` にプランを作成する
- 完了したら `completed/` に移動し、学びを記録する
- 技術的負債を発見したら `tech-debt-tracker.md` に追記する
